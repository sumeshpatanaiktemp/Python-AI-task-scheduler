## Lessons learned

- Always test the Python script from a normal terminal first with the exact command you intend to use in Task Scheduler or the `.bat` file (same interpreter, same paths) [web:22].
- In Task Scheduler, use:
  - Full path to `python.exe` in “Program/script”
  - Full path to the script in “Add arguments”
  - Project directory in “Start in”  
  This pattern greatly reduces “nothing happens” issues [web:18][web:21][web:22].
- A simple `.bat` wrapper is an easy way to control activation of virtual environments, set environment variables, and log output before handing off to Python.
- Clear, explicit documentation (like this development log and a focused README) makes it easier to remember how startup behavior is wired when revisiting the project later or sharing it with others.
- Different AI models have different workings and output formats some may/not produce a particular type of output
- Creation of Log files helps in understanding issues related to the code and helps making debugging easier
